# ຫຼັກການຂອງ LAKI

ເອກະສານນີ້ລະບຸຫຼັກການສໍາລັບ documentation, ຕົວຢ່າງ, review workflow, ແລະວຽກດ້ານຄຸນນະພາບຂໍ້ມູນຂອງ LAKI.

## ພາສາລາວກ່ອນ

LAKI ເລີ່ມຈາກຄວາມຕ້ອງການຂອງພາສາລາວ, ການໃຊ້ງານຂອງຄົນລາວ, ແລະບໍລິບົດລາວ. ພາສາອັງກິດຊ່ວຍໃຫ້ຮ່ວມມືກັບຄົນອື່ນໄດ້, ແຕ່ເປົ້າໝາຍຫຼັກແມ່ນການເສີມພື້ນຖານພາສາລາວ.

## ກວດໂດຍມະນຸດກ່ອນຍອມຮັບ

LAKI ແມ່ນໂຄງການທີ່ນໍາໂດຍມະນຸດ. AI ອາດຊ່ວຍຮ່າງ, ທົດສອບ, ຫຼື ວິເຄາະ, ແຕ່ຜົນງານຈະບໍ່ຖືກຍອມຮັບພຽງເພາະ AI ສ້າງຂຶ້ນ. ຕ້ອງມີການກວດໂດຍມະນຸດ, ການກວດແຫຼ່ງອ້າງອີງ, ແລະການຢືນຢັນ.

## ຄວາມນ່າເຊື່ອຖືຂອງແຫຼ່ງອ້າງອີງສໍາຄັນ

ຂໍ້ມູນຈາກແຫຼ່ງທາງການ, ສະຖາບັນ, ງານວິໄຈ, ປຶ້ມອ້າງອີງ, ບັນທຶກຈາກຜູ້ໃຊ້ພາສາລາວ, AI, ແລະແຫຼ່ງທີ່ບໍ່ຮູ້ທີ່ມາ ບໍ່ຄວນຖືວ່າເທົ່າກັນ.

## ຮັກສາທີ່ມາຂອງຂໍ້ມູນ

ເມື່ອແຫຼ່ງຂໍ້ມູນປອດໄພ ແລະມີສິດໃຊ້, ບັນທຶກ review ຄວນຮັກສາຂໍ້ມູນທີ່ພໍໃຫ້ຮູ້ວ່າກວດຈາກຫຍັງ. ແຕ່ public repo ບໍ່ຄວນເປີດເຜີຍວັດຖຸດິບທີ່ຈໍາກັດສິດ ຫຼືອ່ອນໄຫວ.

## ບໍ່ຮັບຂໍ້ມູນທີ່ AI ສ້າງແບບບໍ່ກວດ

ຂໍ້ມູນທີ່ AI ສ້າງແມ່ນ candidate. ມັນອາດຊ່ວຍເປັນຮ່າງ ຫຼື test case, ແຕ່ຕ້ອງຜ່ານການກວດກ່ອນຈຶ່ງຍອມຮັບ.

## ຊຸດຂໍ້ມູນນ້ອຍທີ່ກວດແລ້ວ ກ່ອນ dataset ໃຫຍ່

LAKI ເລືອກຄຸນນະພາບກ່ອນປະລິມານ. ຊຸດຂໍ້ມູນນ້ອຍທີ່ກວດດີ ແລະມີທີ່ມາຊັດເຈນ ດີກວ່າ dataset ໃຫຍ່ທີ່ບໍ່ຮູ້ຄຸນນະພາບ.

## ບໍ່ໃສ່ຂໍ້ມູນສ່ວນຕົວ ຫຼື ອ່ອນໄຫວໃນ public repo

public repo ນີ້ຫ້າມມີ private code, ຂໍ້ມູນດິບສ່ວນຕົວ, review CSV, finance parser row, dataset ພາຍໃນ, API key, credential, ຂໍ້ມູນສ່ວນຕົວ, ເອກະສານອ່ອນໄຫວ, ຫຼືຂໍ້ມູນຈາກ `data/` folder.

## ບອກຄວາມບໍ່ແນ່ນອນໃຫ້ຊັດ

ຖ້າຍັງບໍ່ໄດ້ກວດ ຕ້ອງລະບຸໃຫ້ຊັດ. ຄວາມບໍ່ແນ່ນອນທີ່ຊື່ສັດປອດໄພກວ່າຄໍາຕອບທີ່ໝັ້ນໃຈແຕ່ບໍ່ມີຫຼັກຖານ.

## ການໃຊ້ງານຈິງໃນທ້ອງຖິ່ນສໍາຄັນ

LAKI ຄວນຊ່ວຍວຽກທີ່ໃຊ້ໄດ້ຈິງ: OCR, search, parsing, ການສຶກສາ, ການວິໄຈ, ແອັບທ້ອງຖິ່ນ, ເອກະສານສາທາລະນະ, ແລະການປະເມີນ AI ທີ່ດີຂຶ້ນ.
