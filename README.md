# LAKI

**Lao AI Knowledge Infrastructure**

LAKI is a Lao-native AI knowledge infrastructure project focused on improving the fundamentals of Lao language data, source validation, OCR evaluation, parser logic, reviewed datasets, and human-validated knowledge pipelines.

LAKI is not trying to replace frontier AI models such as GPT, Claude, Gemini, or other general-purpose systems. LAKI is building a Lao-native data quality layer that can support AI tools, language models, OCR systems, parsers, research datasets, and local applications.

## Why Lao AI Needs Better Fundamentals

Many AI systems can produce Lao text, but Lao output is often uneven. Common problems include translation-like wording, Thai-influenced structure, inconsistent spelling, weak local context, missing source authority, OCR errors, and confident but unsupported local facts.

For Lao AI to become more useful in real work, the foundation needs to improve:

- better reviewed Lao language examples
- clearer source authority and provenance
- safer data-handling rules
- practical OCR and parser evaluation
- human review before public acceptance
- explicit uncertainty instead of overconfident answers

See [docs/LAO_AI_PROBLEM.md](docs/LAO_AI_PROBLEM.md) for more detail.

## What LAKI Focuses On

- Lao-native data quality practices
- source validation and authority tiers
- OCR evaluation methodology for Lao text
- parser logic and topic-pack review workflows
- small reviewed public examples before larger datasets
- benchmark and evaluation formats
- human-validated knowledge pipelines for local use

## What LAKI Is Not

- LAKI is not a claim to be the best Lao AI model.
- LAKI is not a replacement for GPT, Claude, Gemini, or any frontier AI model.
- LAKI is not a place to publish private, sensitive, or unreviewed raw data.
- LAKI is not a guarantee that AI-generated text is correct.
- LAKI is not a marketing project built around hype.

## Human-Led Review Principle

LAKI is human-led. AI tools may assist with drafting, testing, and analysis, but reviewed outputs are not accepted only because they were generated by AI. Final acceptance requires human review, source checking, and validation.

This principle applies to examples, methodology notes, registry entries, parser outputs, OCR outputs, and any future reviewed packs.

## Development Status

This public repository is in early development. It currently contains public documentation, methodology notes, safety principles, and fake examples. It does not publish private datasets, production review data, raw source collections, or sensitive materials.

Public contribution is not fully open yet. For now, LAKI is publishing its principles, methodology, and safety framework before accepting external data contributions.

## Data Safety Note

This public repository is documentation-first. It must not contain private code, private working notes, raw private data, review CSVs, finance parser rows, internal datasets, API keys, credentials, government-sensitive materials, copyrighted raw material, or anything from a `data/` folder.

Only safe public documentation, methodology notes, issue templates, and clearly fake examples belong here. See [docs/DATA_SAFETY_POLICY.md](docs/DATA_SAFETY_POLICY.md).

## Contribution Status

Public contribution is not fully open yet. Do not submit external datasets, raw source material, private examples, personal data, financial records, copyrighted raw material, or government-restricted material.
