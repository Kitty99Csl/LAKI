# LAKI Roadmap
ແຜນພັດທະນາຂອງ LAKI

ແຜນນີ້ເນັ້ນຄວາມປອດໄພ ແລະການໃຊ້ງານຈິງ. ມັນແມ່ນແຜນສໍາລັບ documentation ແລະ methodology ສາທາລະນະ, ບໍ່ແມ່ນການປ່ອຍ dataset ສ່ວນຕົວ.

## Phase 1
ຂໍ້ມູນໂຄງການ ແລະຫຼັກການຄວາມປອດໄພ

ສ້າງ README, Lao README, principles, data safety policy, ຂອບເຂດການ contribution, ແລະ issue templates.

## Phase 2
ແບບຈໍາລອງຄວາມນ່າເຊື່ອຖືຂອງແຫຼ່ງອ້າງອີງ

ກໍານົດລະດັບແຫຼ່ງອ້າງອີງຢ່າງງ່າຍ: ທາງການ, ວິຊາການ, ສິ່ງພິມອ້າງອີງ, community note, AI-generated candidate, ແລະແຫຼ່ງທີ່ບໍ່ຮູ້ທີ່ມາ.

## Phase 3
ເອກະສານບັນຫາຄຸນນະພາບ AI ພາສາລາວ

ອະທິບາຍບັນຫາທີ່ພົບເລື້ອຍ ເຊັ່ນ ພາສາຄ້າຍການແປ, ໂຄງສ້າງຈາກພາສາອື່ນ, ບໍລິບົດທ້ອງຖິ່ນອ່ອນ, ຄໍາສະກົດບໍ່ສອດຄ່ອງ, OCR ຜິດ, ແລະຂໍ້ມູນທ້ອງຖິ່ນທີ່ບໍ່ມີແຫຼ່ງອ້າງອີງ.

## Phase 4
ວິທີປະເມີນ OCR

ສ້າງ methodology ສາທາລະນະເພື່ອທົດສອບ OCR ພາສາລາວຢ່າງປອດໄພ, ລວມທັງປະເພດ error, review status, ແລະຕົວຢ່າງປອມ.

## Phase 5
Parser ແລະ Topic-Pack Review Workflow

ກໍານົດ workflow ສໍາລັບ parser output ແລະ topic pack: ບັນທຶກແຫຼ່ງ, ຈັດ domain, ກວດພາສາລາວ, ກວດ source authority, ກໍານົດ status, ແລະ validate format.

## Phase 6
ຕົວຢ່າງສາທາລະນະທີ່ປອດໄພ

ເຜີຍແຜ່ສະເພາະຕົວຢ່າງປອມທີ່ປອດໄພ ເພື່ອສະແດງ schema, note, validation report, ແລະ review concept.

## Phase 7
Benchmark/Evaluation Format

ສໍາຫຼວດຮູບແບບ benchmark ແລະ evaluation ສໍາລັບ AI output ພາສາລາວ, OCR, parser behavior, ແລະຄໍາຕອບທີ່ອີງແຫຼ່ງອ້າງອີງ.

## Phase 8
Plugin/Export Pack Exploration

ສໍາຫຼວດວ່າ LAKI methodology ສາມາດຮອງຮັບ export pack, plugin, ຫຼື local application integration ໄດ້ຢ່າງປອດໄພຫຼືບໍ່.
