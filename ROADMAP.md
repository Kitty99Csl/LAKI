# LAKI Public Roadmap

This roadmap is intentionally practical and safety-first. It describes public documentation and methodology work, not the release of private datasets.

## Phase 1: Public Project Profile and Safety Principles

Establish the public README, Lao README, principles, data safety policy, contribution boundaries, and issue templates.

## Phase 2: Source Authority Model

Define simple source authority tiers and review notes so contributors can distinguish official, academic, published, community, AI-generated, and unknown sources.

## Phase 3: Lao AI Quality Problem Documentation

Document common Lao AI quality issues such as translation-like wording, Thai-influenced structure, weak local context, spelling inconsistency, OCR errors, and unsupported local facts.

## Phase 4: OCR Evaluation Methodology

Create public methodology for testing Lao OCR outputs safely, including error types, review statuses, and fake examples.

## Phase 5: Parser and Topic-Pack Review Workflow

Define a review workflow for parser outputs and topic packs: source recording, domain classification, Lao wording review, authority checks, status marking, and validation.

## Phase 6: Safe Public Examples

Publish only fake, public-safe examples that demonstrate schemas, notes, validation reports, and review concepts without exposing real private data.

## Phase 7: Benchmark/Evaluation Format

Explore simple benchmark and evaluation formats for Lao AI outputs, OCR, parser behavior, and source-aware answer quality.

## Phase 8: Plugin/Export Pack Exploration

Explore whether LAKI methodology can support safe export packs, plugins, or local application integrations without exposing restricted data.
