# Data Safety Policy

This public repository is for safe documentation, methodology notes, issue templates, and fake examples. It is not a data dump.

## What Must Not Be Published Here

Do not publish:

- raw private data
- sensitive data
- personal data
- financial data
- government-sensitive or government-restricted material
- private operational data
- private working-repo code
- review CSVs or internal review rows
- finance parser rows
- internal datasets
- API keys, credentials, tokens, certificates, or secrets
- copyrighted raw material without permission
- anything from a `data/` folder

## Unreviewed Data Is Not Trusted Data

This repository does not publish unreviewed data as trusted. Any candidate example, OCR output, parser output, or AI-generated text must be marked honestly and reviewed before acceptance.

## Safe Examples Only

Public examples should be fake, minimal, and clearly safe. They should demonstrate structure, fields, statuses, and methodology without exposing real private content.

## Data Acceptance Requirements

Before any data-like material is accepted publicly, it must pass:

1. safety review
2. source rights review
3. source authority check
4. Lao wording review when relevant
5. format validation
6. clear status marking

AI-generated candidate data is never automatically trusted.
