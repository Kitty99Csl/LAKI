# Review Workflow
ຂັ້ນຕອນການກວດຂອງ LAKI

workflow ນີ້ອະທິບາຍເສັ້ນທາງທີ່ປອດໄພຈາກ candidate text ໄປຫາ accepted public example. ມັນຖືກອອກແບບສໍາລັບ documentation ແລະຕົວຢ່າງປອມໃນ repository ນີ້.

## Steps
ຂັ້ນຕອນ

1. collect candidate
2. record source
3. classify domain
4. review Lao wording
5. check source authority
6. mark status
7. validate format
8. accept only after review

## Statuses
ສະຖານະ

| Status | ຄວາມໝາຍ |
| --- | --- |
| candidate | item ໃໝ່. ຍັງບໍ່ເຊື່ອຖື. |
| needs_review | ຕ້ອງການການກວດໂດຍມະນຸດກ່ອນໃຊ້. |
| reviewed | ກວດໂດຍມະນຸດແລ້ວ, ແຕ່ອາດຍັງບໍ່ໄດ້ຮັບ final acceptance. |
| corrected | item ຖືກແກ້ຫຼັງ review. ຄວນຮັກສາ correction history ເທົ່າທີ່ປອດໄພ. |
| rejected | item ບໍ່ຄວນໃຊ້. |
| accepted_public_example | ຕົວຢ່າງປອມ ຫຼື public-safe example ທີ່ຍອມຮັບໃນ repository ນີ້. |

## Review Checklist
Checklist ສໍາລັບ Review

- ບັນທຶກແຫຼ່ງແລ້ວບໍ?
- ຈັດ source type ແລ້ວບໍ?
- ປອດໄພທີ່ຈະເຜີຍແຜ່ບໍ?
- ຫຼີກລ້ຽງຂໍ້ມູນສ່ວນຕົວ, ລັບ, ອ່ອນໄຫວ, ການເງິນ, ຫຼືຈໍາກັດສິດບໍ?
- ພາສາລາວເປັນທໍາມະຊາດ ແລະເໝາະສົມບໍ?
- factual claims ຜ່ານ source-check ແລ້ວບໍ?
- ລະບຸ uncertainty ແລ້ວບໍ?
- item ກົງກັບ schema ຫຼື format ທີ່ຕ້ອງການບໍ?

## Acceptance Rule
ກົດການຍອມຮັບ

ຢ່າຍອມຮັບຂໍ້ມູນພຽງເພາະມັນອ່ານລື່ນ, AI ສ້າງ, ຫຼື format ຖືກ. ການຍອມຮັບຕ້ອງມີ review, source checking, safety screening, ແລະ validation.
