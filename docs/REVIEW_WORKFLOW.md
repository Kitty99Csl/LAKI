# Review Workflow

This workflow describes a safe path from candidate text to accepted public example. It is designed for documentation and fake examples in this repository, and can later inform stricter review systems.

## Steps

1. Collect candidate
2. Record source
3. Classify domain
4. Review Lao wording
5. Check source authority
6. Mark status
7. Validate format
8. Accept only after review

## Statuses

| Status | Meaning |
| --- | --- |
| candidate | Newly collected item. Not trusted yet. |
| needs_review | Requires human review before use. |
| reviewed | Human review has happened, but final acceptance may still be pending. |
| corrected | Item was edited after review. Correction history should be preserved where safe. |
| rejected | Item should not be used. |
| accepted_public_example | Safe fake or public-safe example accepted for this repository. |

## Review Checklist

- Is the source recorded?
- Is the source type classified?
- Is the material safe to publish?
- Does it avoid personal, private, sensitive, financial, or restricted content?
- Is the Lao wording natural and appropriate?
- Are factual claims source-checked?
- Is uncertainty marked?
- Does the item match the expected schema or format?

## Acceptance Rule

Do not accept data only because it is fluent, generated by AI, or formatted correctly. Acceptance requires review, source checking, safety screening, and validation.
